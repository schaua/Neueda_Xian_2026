# Mini Spring Development Project

## The Smartphone Apps Project

Create a new project, and within the Project use Spring to create an application represent the REST backend of an Smartphone App Store application.

Create classes to represent the various necessary entities and connect to a database to presist the information between application runs.  

A phone user should be able to browse available apps by name and by category.

A phone user should be able to purchase an available app and it will no longer show up on the list of available apps.

A phone user should be able to list the apps that they have already purchased.

Pay particular attention to:

1. Appropriate application of concepts such as a 3-tier architecture
2. Appropriate use of dependency injection
3. Appropriate use of interfaces
4. Appropriate use of Spring modules, annotations and configuration
6. Readable and maintainable code
7. A simple readme.md document specifying any design decisions and assumptions that you have made as you have implemented this project.

You will have 20 minutes per team this afternoon (starting at 3 PM) to present your project to your peers and instructors (in English please).

Bonus points if every member of the team speaks during the presentation.
